# 📋 Менеджер задач (Task Manager Service)

## 📋 О проекте
Веб-приложение для помощи пользователям в организации рабочего процесса, управлении задачами и соблюдении дедлайнов. Позволяет создавать задачи, назначать приоритеты, отслеживать статус выполнения и анализировать продуктивность.

## 🎯 Цель разработки
Разработать концепцию и техническое задание для веб-приложения «Менеджер задач».

## 📁 Структура репозитория
- `/mockups` - макеты интерфейса (изображения)
- `/docs` - документация проекта
- `README.md` - основной файл с техническим заданием

## 1. АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ

### 1.1 Описание предметной области
В условиях высокой загрузки и множественных дедлайнов пользователям необходимо эффективно планировать задачи, контролировать их выполнение и анализировать продуктивность. Без системного инструмента важные дела теряются, сроки срываются, а приоритеты путаются.

**Проблемы текущей ситуации:**
- Отсутствие системного подхода к планированию дел
- Пропуск важных дедлайнов и встреч
- Отсутствие визуализации загрузки и приоритетов
- Хаос в заметках и бумажных ежедневниках

### 1.2 Целевая аудитория
| Роль | Потребности |
|------|------------|
| **Пользователь** | Вести учёт задач, видеть дедлайны, получать мотивацию |
| **Премиум-пользователь** | Расширенная аналитика, командные проекты, неограниченное число задач |
| **Администратор** | Управление пользователями, модерация контента |

### 1.3 Решаемые задачи
- Автоматизация учёта выполнения рабочих задач
- Визуализация прогресса по проектам
- Анализ продуктивности по категориям задач
- Централизованное хранение истории изменений статусов

## 2. ФОРМАЛИЗАЦИЯ ТРЕБОВАНИЙ

### 2.1 Функциональные требования

#### Роль «Пользователь»
| ID | Требование | Описание |
|----|-----------|----------|
| F01 | Авторизация | Вход в систему по логину/паролю или через Google |
| F02 | Управление задачами | Создание, редактирование, удаление задач (название, описание, приоритет, срок) |
| F03 | Смена статуса | Отметка задачи как выполненной, в процессе или отложенной |
| F04 | История изменений | Просмотр логов изменения статусов за прошлые периоды |
| F05 | Статистика | Просмотр графиков успешности и закрытых задач за период |
| F06 | Настройки профиля | Изменение пароля, имени, настроек уведомлений |

#### Роль «Администратор»
| ID | Требование | Описание |
|----|-----------|----------|
| A01 | Управление пользователями | Блокировка нарушителей, просмотр списка пользователей |
| A02 | Модерация | Проверка контента (если есть публичные проекты) |
| A03 | Системная статистика | Общая активность пользователей на платформе |

### 2.2 Нефункциональные требования

| ID | Требование | Описание | Критерий |
|----|-----------|----------|----------|
| N01 | Производительность | Время загрузки дашборда | ≤ 1.5 секунды |
| N02 | Надёжность | Сохранение данных задач | 100% сохранность записей |
| N03 | Доступность | Работа через браузер и PWA | Chrome, Safari, Mobile |
| N04 | Адаптивность | Корректное отображение на смартфонах | Да (Mobile First) |
| N05 | Безопасность | Хранение паролей в хешированном виде | Обязательно (bcrypt) |
| N06 | Интуитивность | Смена статуса задачи в 1 клик | ≤ 1 клик с дашборда |

## 3. ТЕХНОЛОГИЧЕСКИЙ СТЕК

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| **Frontend** | React | Компонентный подход, идеален для интерактивных дашбордов |
| **UI Library** | Ant Design / Chakra UI | Готовые компоненты, темизация, адаптивность |
| **State Manager** | Redux Toolkit | Управление глобальным состоянием (список задач, пользователь) |
| **Backend** | Node.js + Express | Быстрая разработка API, единый язык с фронтендом |
| **Database** | PostgreSQL | Надёжное хранение связей пользователь-задача-лог |
| **ORM** | Sequelize | Упрощение работы с БД, валидация данных |
| **API** | REST | Стандартный подход для взаимодействия клиент-сервер |
| **Version Control** | Git + GitHub | Контроль версий, портфолио |
| **Hosting** | Vercel (frontend) + Render (backend) | Удобный деплой для учебных проектов |

## 4. МОДЕЛЬ ДАННЫХ

### 4.1 ER-диаграмма (описание связей)
```
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│     Users       │       │     Tasks       │       │    TaskLogs     │
│   (Пользователи)│       │    (Задачи)     │       │ (История статусов)│
├─────────────────┤       ├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────┤ userId (FK)     │       │ id (PK)         │
│ email           │       │ title           │       │ taskId (FK)     │
│ passwordHash    │       │ priority        │       │ oldStatus       │
│ fullName        │       │ dueDate         │       │ newStatus       │
│ createdAt       │       │ status          │       │ createdAt       │
│                 │       │ createdAt       │       │                 │
└─────────────────┘       └─────────────────┘       └─────────────────┘
```

### 4.2 Описание таблиц

**Таблица: Users (Пользователи)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| email | VARCHAR(100) | Email пользователя | UNIQUE, NOT NULL |
| passwordHash | VARCHAR(255) | Хеш пароля | NOT NULL |
| fullName | VARCHAR(100) | Полное имя | NOT NULL |
| createdAt | TIMESTAMP | Дата регистрации | DEFAULT NOW() |

**Таблица: Tasks (Задачи)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| userId | INTEGER | ID владельца | FOREIGN KEY (Users.id) |
| title | VARCHAR(100) | Название задачи | NOT NULL |
| priority | ENUM | Приоритет (low/medium/high) | DEFAULT 'medium' |
| dueDate | DATE | Срок выполнения | NOT NULL |
| status | ENUM | Статус (new/process/done) | DEFAULT 'new' |
| createdAt | TIMESTAMP | Дата создания | DEFAULT NOW() |

**Таблица: TaskLogs (История статусов)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| taskId | INTEGER | ID задачи | FOREIGN KEY (Tasks.id) |
| oldStatus | VARCHAR(20) | Предыдущий статус | NOT NULL |
| newStatus | VARCHAR(20) | Новый статус | NOT NULL |
| createdAt | TIMESTAMP | Время изменения | DEFAULT NOW() |

**Проверка:**
- Нельзя создать задачу с пустым названием
- `dueDate` ≥ сегодня при создании — нельзя ставить дедлайн в прошлом
- CASCADE при удалении пользователя — удаляются его задачи и логи

## 5. ПРОЕКТИРОВАНИЕ API

### 5.1 REST API Endpoints

#### 🔐 **Аутентификация**
| Метод | Endpoint | Описание | Тело запроса | Ответ |
|-------|---------|----------|--------------|-------|
| POST | `/api/auth/login` | Вход в систему | `{ "email": "user@mail.com", "password": "***" }` | `{ "token": "jwt_token", "user": {...} }` |
| POST | `/api/auth/logout` | Выход | - | `{ "success": true }` |
| POST | `/api/auth/register` | Регистрация | `{ "email": "...", "password": "...", "fullName": "..." }` | `{ "token": "...", "user": {...} }` |

#### 📋 **Задачи (Tasks)**
| Метод | Endpoint | Описание | Параметры/Тело | Ответ |
|-------|---------|----------|---------------|-------|
| GET | `/api/tasks` | Список моих задач | - | Массив задач |
| GET | `/api/tasks/:id` | Детали задачи | - | Объект задачи |
| POST | `/api/tasks` | Создать задачу | `{ "title": "Отчёт", "priority": "high", "dueDate": "2026-01-20" }` | Созданная задача |
| PUT | `/api/tasks/:id` | Обновить задачу | `{ "title": "Отчёт за месяц" }` | Обновлённая задача |
| DELETE | `/api/tasks/:id` | Удалить задачу | - | `{ "success": true }` |

#### 🔄 **Логи изменений (Logs)**
| Метод | Endpoint | Описание | Параметры/Тело | Ответ |
|-------|---------|----------|---------------|-------|
| POST | `/api/logs` | Сменить статус | `{ "taskId": 5, "status": "done" }` | `{ "success": true, "task": {...} }` |
| DELETE | `/api/logs/:id` | Отменить изменение | - | `{ "success": true }` |
| GET | `/api/logs/history` | История по задаче | `?taskId=5&period=month` | Массив записей |

#### 📊 **Статистика (Stats)**
| Метод | Endpoint | Описание | Параметры | Ответ |
|-------|---------|----------|-----------|-------|
| GET | `/api/stats` | Общая статистика | `?userId=5&period=month` | `{ "completed": 20, "total": 30, "rate": 66.7 }` |
| GET | `/api/stats/priority` | По приоритетам | `?userId=5` | `{ "high": 5, "medium": 10, "low": 15 }` |
| GET | `/api/stats/deadlines` | Дедлайны | `?userId=5` | `{ "overdue": 2, "upcoming": 5 }` |
